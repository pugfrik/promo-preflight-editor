# Measurement readiness — the pre-registration checklist

Used by the **Measurability** lens. A promo is measurable when ALL of
these exist BEFORE it runs — afterward is too late; post-hoc numbers are
cherry-picking.

- [ ] **Baseline exists.** Current state measured before launch: a
  trailing same-weekday window from the POS (8 weeks is a sound
  default), or — where the POS can't see the metric — a manual count the
  operator signs off on. Never guessed, never remembered.
- [ ] **Metric named and reachable.** The specific number that will move
  (covers, item sales, redemptions, sign-ups) and confirmation the POS —
  or a defined manual process — can actually tag it.
- [ ] **1–3 numeric targets, time-boxed, pre-registered.** Each written
  as the claim it would earn, at a tier the run length supports (see
  `claim-tiers.md`).
- [ ] **Enough data to read a signal.** Rough check: if the baseline
  window averages a handful of covers, a one-week promo can't produce a
  readable delta — extend the run or pick a busier metric.
- [ ] **Attribution path.** How will a redemption be tied to the promo
  (code, ticket, member check-in, time window)? "We'll just see if we're
  busier" is not attribution.

Pre-registration is what turns a hit into a credible before/after story
instead of a post-hoc cherry-pick — and turns a miss into a lesson
instead of a mystery.
